# LeftPad

スマホを左手デバイス（ショートカットパッド）として使い、PCへショートカットキーを送信するローカルアプリ。

## 構成

- `server.py`（PC側）
  - HTTPサーバー（`8080`）で `smartphone.html` を配信
  - WebSocketサーバー（`8765`）でスマホからの操作コマンドを受信
  - `pyautogui` でPCのキー入力を実行
  - Tkinter GUIでQRコード表示・接続状態表示・ショートカット編集
- `smartphone.html`（スマホ側）
  - 描画 / 表示 / レイヤー / ジェスチャー / カスタム / 設定 タブを提供
  - WebSocket通信でコマンド送信
- `shortcuts.json`
  - コマンド名 → キー配列（`ctrl+z` など）の定義
- `gestures.json`
  - ジェスチャー名 → コマンド（または `ctrl+s` 形式の生キー）の定義

## セットアップ

```bash
pip install -r requirements.txt
python server.py
```

Windows は `run.bat` から起動可能。

## 使い方

1. PCで `python server.py` を起動
2. 表示されたQRコードをスマホで読み取る
3. スマホ画面で自動接続（または `IP:8765` を入力して接続）
4. ボタン操作・ジェスチャーでPCへショートカット送信

## 認証方式（2026-04 変更）

### 背景

過去実装では `ws://<host>:8765?token=...` の**URLクエリ**で認証トークンを渡していた。
しかし `websockets` ライブラリのバージョン差異や実行環境によって、サーバー側で取得する `path/request_uri` が想定どおりでないケースがあり、
**QRで正しくアクセスしても認証失敗（4001）**が起きることがあった。

### 現在の実装

- QRコードで開くURL（HTTP）にはトークンを付与（`smartphone.html?token=...`）
- WebSocket URLにはトークンを付与しない（`ws://<host>:8765`）
- 接続直後、クライアントが最初のメッセージとして以下を送信

```json
{ "type": "auth", "token": "..." }
```

- サーバーは最初のメッセージを検証し、成功時のみ以降のコマンド受信を許可

```json
{ "type": "auth", "ok": true }
```

失敗時は `4001 Unauthorized` で切断。

この方式により、URLパス解析仕様に依存せず、認証の安定性が向上。

## トラブルシューティング

- **「認証失敗：QRから再アクセス」**が出る
  - 古いタブを再利用せず、最新のQRから開き直す
  - サーバー再起動後はトークンが変わるため、必ず再読取する
- 接続できない
  - スマホとPCが同一ネットワークか確認
  - ファイアウォールで `8080` / `8765` を許可
- キー入力が効かない
  - 対象アプリがアクティブか確認
  - `shortcuts.json` のキー名を `pyautogui` 対応キーに合わせる

## 追加機能（2026-04）

- PC側「ショートカット編集」ウィンドウはデフォルト全画面で起動
- ショートカット割り当ては文字入力ではなく「キーを記録」で実キー取得
- PC側からショートカットの追加が可能
- PC側からジェスチャー割り当て（`gestures.json`）を編集可能
- ジェスチャー拡張:
  - 二本指タップ / 二本指スワイプ（上下左右）
  - 三本指タップ / 三本指スワイプ（上下左右）
  - 画面ホールド
- スマホ側にボタン位置調節モードを追加（ドラッグで自由配置）
- スマホ側のタブ・ボタンラベルを大きく太く調整
